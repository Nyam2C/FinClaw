# README Section Conventions

FinClaw README.md 작성의 일관성을 위한 규약. author 는 이 문서를 읽고 합성한다.

## 톤

- **1 인칭 복수 금지** ("우리는...", "we ..."). FinClaw 는 1 인 사용자 프로젝트. 객관 서술.
- **마케팅 어휘 금지** ("혁신적", "최첨단", "powerful"). 사실만.
- **한국어 primary**, 식별자(파일명, 클래스명, 환경변수, 명령어)는 영어 그대로. 줄 안에서 자연스럽게 혼용.
- **존댓말/반말:** 평서형(\"~한다\", \"~이다\"). 사용자 메모리 문체와 일치.

## 구조

### 헤딩 레벨

- `# FinClaw` — 문서 제목, 1 회만.
- `## ` — 최상위 섹션 (빠른 시작, 기능, 아키텍처, ...).
- `### ` — 하위 섹션. `####` 이상은 가급적 피하고 표/리스트로 대체.

### 권장 섹션 순서

```
# FinClaw
> One-liner

Elevator pitch (1 paragraph)

## 왜 FinClaw 인가 (차별점 3 개, bullet)

## 빠른 시작 (5 분)
## 기능 (Feature Tour)
## 아키텍처
## 패키지 구조
## 환경변수
## 설정 파일 (있는 경우)
## 운영
  ### 테스트
  ### Docker
  ### 트러블슈팅
## 보안 (인증/JWT 등)
## 기여 (개발 워크플로우)
## 라이선스 (있는 경우)
```

섹션을 추가/삭제하는 것은 허용. 단, 위 순서는 30-초 / 5-분 / 30-분 정보 깊이 원칙을 만족하므로 가능한 유지.

## 시각 요소

### 코드 블록

- 언어 라벨 필수: ` ```bash `, ` ```typescript `, ` ```mermaid `, ` ```json ` 등.
- 출력 예시는 ` ```text ` 또는 라벨 없이.
- 명령어는 prompt 기호 (`$`, `>`)를 붙이지 않는다 — 복사·실행 편의를 위해.

### 표

- 정렬: 좌(`---`), 중앙(`:---:`), 우(`---:`). 환경변수 표는 좌 정렬 통일.
- 환경변수 표 표준 컬럼: `| 변수 | 설명 | 기본값/예시 | 비고 |`.
- 패키지 표 표준 컬럼: `| 패키지 | 역할 | 의존 (workspace) | 외부 surface |`.
- 명령어 표 표준 컬럼: `| 명령어 | 설명 |` (간단 시) 또는 `| 명령어 | 설명 | 언제 쓰나 |` (상세).

### 다이어그램

- mermaid 우선 (GitHub 렌더링 지원).
- 패키지 의존 그래프: `graph TD` 또는 `graph LR`.
- 데이터 흐름은 mermaid `sequenceDiagram` 또는 번호 매긴 텍스트 리스트.
- 너무 복잡하면 핵심만 다이어그램으로, 부속은 표로.

### 인용 박스

- `>` 인용은 한 줄 강조 (one-liner 직후, 또는 함정 경고).
- 함정 경고는 `> ⚠️ **경고**:` 패턴 (이모지 1 개 허용 — 시인성).

## 길이

- README.md 전체: **400~700 줄 목표**, 최대 1000 줄.
- 한 섹션: 100 줄 이하 권장. 초과 시 하위 섹션 분리 또는 별도 문서 링크.
- elevator pitch: 200 자 이내, 3 문장 이하.
- 차별점 bullet: 3~5 개, 각 1~2 줄.

## 배지

- 사용 자제. 사용한다면 다음만:
  - Node.js 버전 요구 (실측 버전 — `>= 22.0.0`)
  - License (있는 경우)
  - CI status (실제 CI 가 있을 때만 — 추측 금지)
- shields.io 등 외부 의존 배지는 PR 마다 깨질 수 있으므로 검증된 것만.

## 링크

- 외부 링크: `[표시명](URL)` — URL 은 직접 입력하지 않고 코드/플랜/공식 문서에 있는 것만.
- 내부 링크 (저장소 내 파일): 상대경로 — `[CLAUDE.md](./CLAUDE.md)`.
- 깨진 링크 금지 — 링크하는 파일이 실제 저장소에 존재해야 한다.

## 코드 인용

- 파일 인용: `` `packages/server/src/main.ts` `` (백틱).
- 라인 범위: `` `pipeline.ts:42-58` ``.
- 함수/심볼: `` `MemoryCaptureStage` `` (백틱, 코드블록 X).

## 금지 사항

- TODO/FIXME 가 있는 README 금지 — 미완성 영역은 explicit 으로 "구현 예정 (Phase 27)" 형태로.
- 죽은 링크 금지.
- 추측한 환경변수/명령어 금지 — 코드에 없으면 README 에 없다.
- 이모지 남용 금지 (인용 박스 경고 외).
- 광고성 표현, 가식적인 자랑 금지.
